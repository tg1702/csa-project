Using J-K Flip-flops to Build Bidirectional Synchronous Counters

Objectives

On completion of this section, you should be able to:

  • Understand the function of a bidirectional synchronous counter
  • Use J-K Flip-flops to design a bidirectional synchronous counter

What's a Bidirectional Synchronous Counter?

Simply put, a synchronous counter uses a clock pulse to trigger each flip-flop used in the counter at the same time. A bidirectional counter is a synchronous up/down binary counter that has the ability to count in both directions either to or from some preset value as well as zero. We'll use the 4-step process to help us in desgining our counter.

If you're unfamiliar with J-K flip-flops, take a look here then come back to this page: What are J-K flip flops?

The 4-Step Process

Although bidirectional synchronous counters take a bit more work, we can use the 4-step process from the section that dealt with designing synchronous counters. Let's look at the following example:

Design a 3-bit bidirectional synchronous counter with the following repeated binary sequence:0,1,4,6,7,5 using J-K flip-flops

Step 1: Develop the State Diagram

From previous sections, you should be familiar with how this is done. You might come up with something like this:

Fig. 1: Supposed State Diagram of the binary sequence 0,1,4,6,7,5

Here, we used the binary representation of each number of the sequence. The example tells us we are designing a 3-bit counter. So, we use three bits to represent each number in the sequence.

However, we need more information when working with bidirectional counters. As the word 'bidirectional' suggests, we're moving in two directions. We consider the directions for the 'up' count (M=0) and the 'down' count (M=1).

Fig. 2: Actual State Diagram of the binary sequence 0,1,4,6,7,5

As you can see, the 'up' count goes in a clockwise direction (0,1,4,...) while the 'down' count moves counter-clockwise (0,5,7,6...). This is the actual state diagram of this sequence.

Step 2: Develop the Present-State Next-State Transition Table

In this step, we utilize the excitation table to complete our Present-State Next-State Transition Table. Recall the excitation table for J-K flip-flops.

Fig. 3: Excitation Table for J-K Flip-flop

Looking at our state diagram, we see we move from 0 to 1, 1 to 4, 4 to 6 and 6 to 7 for M = 0. For M = 1, we move from 0 to 7, 7 to 6, 6 to 4 and so on. This shows our present state and next state. With this information, we can create a table that will help us develop the Present-State Next-State Transition Table:

Fig. 4: Present and Next State of the sequence 0,1,4,6,7,5

Using fig. 4 and the excitation table, we create the Present-State Next-State Transition Table

Fig. 5: Present-State Next-State Transition Table

Let's break down how we got this table by looking at the first row.

In this row since M=0, we move from state 000 to 001 and we labeled each bit (as you can see, QA is 0, QB is 0 and QC is 0 for the present state and Q+A is 0, Q+B is 0 and Q+C is 1 for the next state ).

Forget about M for now and focus on the present state QA and its next state Q+A. Notice that both QA and Q+A are 0. Take a look back at the excitation table and see that two 0's gives a 0 for J and X for K. These are the values of the inputs JA and KA

Let's look at QB and Q+B. Again, we have two zeros which gives the inputs JB = 0 and KB = X.

Follow this pattern and check for yourself that JC is 1 and KC is X (you should see QC is 0 and Q+C is 1 which according to the excitation table is J = 1 and K = X).

Following these steps gives the resulting inputs for each row.
